2. Pipeline trước & sau

2.1 Pipeline TRƯỚC (PyMuPDF)

1 PDF → PyMuPDF extract per-page raw text + font metadata
2 Math-aware reconstruction: fraction bars, subscripts, operator remap
3 VLM fallback: scanned/empty pages → GPT-4o-mini transcribe
4 Vietnamese normalization: clean_artifacts, normalize_structure
5 ChunkBlocks(): sentence-split, atomic formulas, max_chars=1000, overlap=200
6 Dense embedding: text-embedding-3-small trên title › section › text
7 BM25: tokenize tiếng Việt, index cùng text
8 Hybrid retrieval: BM25 ∪ Dense → RRF → Rerank

2.2 Pipeline SAU (Reducto agentic)

1 PDF → Reducto API (mode=agentic): VLM review text + tables
2 Reducto chunking: page_sections, size=1200, filter Header/Footer/Page Number
3 Mỗi chunk trả về content (đầy đủ) + embed (tóm tắt tối ưu cho embedding)
4 Vietnamese normalization: text và embed_text đều được normalize
5 BỎ QUA ChunkBlocks() — Reducto đã chunk sẵn theo section
6 Dense embedding: text-embedding-3-small trên title › section › embed_text
7 BM25: tokenize tiếng Việt, index trên title › section › text (đầy đủ)
8 Hybrid retrieval: BM25 ∪ Dense → RRF → Rerank
3 thay đổi quan trọng nhất:
① Dual-channel index: BM25 dùng text (đầy đủ, keyword match), Dense dùng embed_text (tóm tắt, semantic match)
② Bỏ chunk_blocks() thủ công → dùng Reducto page_sections (tôn trọng section boundary)
③ Công thức toán LaTeX được giữ nguyên bởi VLM review trong mode agentic

3.3 So sánh công thức thực tế

S(c) — Chỉ số an toàn

❌ PyMuPDF (trước):
24 - Nobs(c) dmin (C, Oc)
S(c) = C₁ + (1 - C₁). (4)
24 3

→ Phân số hoàn toàn hỏng — tử/mẫu nằm trên cùng 1 dòng
✓ Reducto agentic (sau):
\[ S(c)=C_{1} \cdot \frac{24-N_{\mathrm{obs}}(c)}{24}+\left(1-C_{1}\right) \cdot \frac{d_{\min }\left(c, \mathcal{O}_{c}\right)}{3} \]

→ LaTeX chính xác, phân số rõ ràng

D_ij — Ma trận chi phí (2 phases of optimization)

❌ PyMuPDF (trước):
Di,j = min w1 length(∑ij) + w2 ∫(∑ij) + w3 Risk(∑ij)
→ Ký tự失常: ∑ thay P, ∫ thay R, subscripts thành inline
✓ Reducto agentic (sau):
D_{i,j} = \min w₁·\operatorname{length}(P_{ij}) + w₂·R(P_{ij}) + w₃·\operatorname{Risk}(P_{ij})
s.t. cx_i ∈ {1,...,W̃}, cy_i ∈ {1,...,H̃}
w₁ + w₂ + w₃ = 1

→ LaTeX hoàn chỉnh, subscripts đúng

6. Fallback Strategy

if REDUCTO_PARSE == "agentic" and REDUCTO_API_KEY:
    → Reducto agentic (2.5 cr/page avg, VLM text+table)
elif REDUCTO_PARSE == "default" and REDUCTO_API_KEY:
    → Reducto default (1 cr/page) — ⚠️ hỏng công thức phân số
else:
    → PyMuPDF + VLM fallback (local, free)

# Nếu Reducto fail (network, rate limit):
    → Tự động fallback sang PyMuPDF

8. Kết luận

  1. Reducto agentic là bắt buộc cho tài liệu có công thức toán — PyMuPDF mất cấu trúc phân số, sai ký tự Hy Lạp (2/5 vs 5/5)
  2. Dual-channel indexing (BM25=text, Dense=embed_text) cải thiện semantic search mà không mất keyword match
  3. Bỏ chunk_blocks() khi dùng Reducto — page_sections tôn trọng section boundary tốt hơn
  4. Chi phí: 20 credits/8 trang; free tier 1000 credits ≈ 50 tài liệu tương đương
  5. Fallback sang PyMuPDF vẫn hoạt động khi REDUCTO_PARSE=off hoặc API fail
